Your task is to analyze the customer complaint below and return a precise classification.

COMPLAINT:
{complaint}

---

CLASSIFICATION RULES:

CATEGORY - choose exactly one:
- BILLING    : incorrect charges, refund requests, invoice disputes, payment failures,
               unauthorized transactions, fraud, duplicate charges
- TECHNICAL  : app errors, login failures, system outages, feature bugs,
               slow performance, data not loading, password reset failures,
               any system failure that prevents the user from completing an action
- DELIVERY   : late delivery, missing order, wrong item received, damaged package
- ACCOUNT    : account locked, password reset requests (not technical failures),
               personal data updates, account closure, suspicious logins,
               phishing reports, possible data exposure
- OTHER      : general feedback, praise, questions that do not describe a problem

URGENCY - choose exactly one based on business and financial impact:
- LOW      : general question, minor inconvenience, no financial impact,

- MEDIUM   : service degraded, customer waiting for resolution,
             account closure or GDPR deletion requests (regulatory obligation),
             wrong or damaged item received
- HIGH     : financial loss reported, service completely unavailable,
             financial data inconsistency between systems,
             delivery not received past the expected date,
             deadline at risk (e.g. payment due soon and system blocks access)
- CRITICAL : fraud suspected, unauthorized transactions reported,
             data breach possible, suspicious login from unknown location,
             phishing attempt reported

SUGGESTED TEAM - choose exactly one from this list (use the exact text):
- "Billing Support Team"
- "Technical Support Team"
- "Logistics Team"
- "Account Management Team"
- "General Support Team"

REASONING - write exactly one sentence explaining your classification decision.

---

IMPORTANT RULES:
- Unauthorized transactions and fraud are always BILLING + CRITICAL.
- If a technical failure is blocking a financial action, urgency is HIGH.
- Financial data inconsistency between platforms is always HIGH urgency.
- Slow performance or minor bugs with no financial impact are TECHNICAL + LOW.
- Account closure and GDPR deletion requests are always MEDIUM.
- When in doubt between two urgency levels, always choose the higher one.
- Base your decision only on what is written in the complaint.
